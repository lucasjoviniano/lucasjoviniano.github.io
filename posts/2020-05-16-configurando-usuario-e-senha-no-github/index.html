Configurando Usuário e Senha no Git/Github

Toda vez que fazemos um push num repositório pro Github precisamos informar o email e a senha. Com o tempo, isso se torna um trabalho tedioso e, como bons programadores que somos, seria uma boa automatizar esse processo. Sendo assim, vou ensinar como configurar chaves SSH para fazer a autenticação.

Criando a Chave SSH

O primeiro passo é criar a chave SSH localmente. No terminal, utilize o comando

ssh-keygen -t rsa -b 4096 -C "email_do_github"

Substituindo o email_do_github pelo endereço de email da sua conta. O comando irá solicitar um nome para a chave e uma senha (que não precisa ser a mesma do Github).

Adicionando a Chave no Sistema

Com a chave em mãos, devemos colocá-la no ssh-agent para que o computador a detecte automaticamente.

eval $(ssh-agent)
ssh-add <NOME_DA_CHAVE>

Substiuindo o nome da chave pelo nome que você deu anteriormente.

Adicionando a Chave ao Github

Acesse esse link. No topo da tela clique em New SSH Key

No camplo Title, adicione um título que identifique o computador. No campo Key, copie e cole o conteúdo do arquivo <NOME_DA_CHAVE>.pub.

Depois é só clicar em Add SSH key.

Mudando a URL do Repositório

Quando fazemos um clone do repositório a opção padrão do github é um link HTTPS. Para podermos usar a chave SSH automaticamente devemos alterar a url para SSH. Você pode conferir a URL com

git remote -v

Para alterar a URL, devemos ir na página do repositório, clicar em Clone or dowload e em Use SSH. Após copiar o link SSH, usar o comando:

git remote set-url origin <URL>

Substituindo pela URL copiada.

E pronto! Agora salvamos alguns cliques digitando email e senha toda vez que precisamos dar um push.
